# Скрипт установки VLESS REALITY + Nginx

Скрипт `install.sh` поднимает VPN-сервер на базе Xray с VLESS REALITY и дополнительно запускает `nginx` для обычного веб-сайта на `80/443`.

Тестировалось на Ubuntu Server 22.04. На других Ubuntu-подобных системах может заработать, но это не гарантируется.

## Быстрая установка

1. Скопируйте репозиторий на сервер.
2. Запустите скрипт от `root`:

```bash
sudo bash install.sh
```

Или скачайте скрипт напрямую с GitHub и сразу запустите его:

```bash
curl -fsSL https://raw.githubusercontent.com/nickolsky/vless_docker_install_scripts/refs/heads/main/install.sh | sudo bash
```

3. Ответьте на вопросы скрипта:
   - `REALITY_DOMAIN` - домен, который Xray будет имитировать, например `dl.google.com`
   - `XRAY_PORT` - порт для Xray, по умолчанию `8443`

два поля ниже оставьте пустыми, они нужны только если вы хотите еще поднять свой web-сервер на том же сервере где у вас работает VPN с какой-либо информацией

   - `NGINX_DOMAIN` - ваш домен, направленный на этот сервер; если оставить пустым, будет только HTTP, оста
   - `LE_EMAIL` - email для Let's Encrypt; если пусто, сертификат не выпускается

4. После завершения скрипт выведет готовые параметры подключения:
   - IP-адрес сервера
   - порт
   - UUID
   - Public Key
   - Short ID
   - SNI

## Что делает скрипт

Скрипт автоматизирует установку и запуск Xray в режиме `VLESS + REALITY` через Docker.

В процессе он:

- проверяет, что запуск идет от `root`
- проверяет наличие Docker и Docker Compose Plugin, а при необходимости устанавливает их
- генерирует `UUID`, REALITY-ключи и `shortId`
- создает конфигурацию Xray в `/opt/xray`
- запускает контейнер `xray-reality`
- создает простую HTML-страницу в `/opt/www/html`
- полностью пересоздает каталог `/opt/nginx`
- создает Docker Compose-конфигурацию для `nginx` и `certbot`
- сначала поднимает `nginx` только по HTTP
- при наличии домена и email пытается выпустить сертификат Let's Encrypt и включить HTTPS
- открывает порты в firewall: `22/tcp`, `XRAY_PORT/tcp`, `80/tcp`, `443/tcp`
- в конце показывает итоговые параметры клиента и статус контейнеров

## Что важно знать заранее

- Скрипт рассчитан на запуск на чистом или почти чистом сервере.
- Если на сервере уже работают `nginx` или `apache2`, скрипт остановит и отключит их, чтобы освободить `80/443`.
- Каталог `/opt/nginx` удаляется и создается заново.
- Если указанный порт Xray уже занят, скрипт автоматически попробует следующий свободный порт.
- Для выпуска сертификата Let's Encrypt домен `NGINX_DOMAIN` должен указывать на этот сервер.
- `NGINX_DOMAIN` должен быть именно вашим доменом, а не доменом-маскировкой REALITY.
- Если сертификат не получится выпустить, `nginx` останется работать только по HTTP.

## Какие данные спросит скрипт

### `REALITY_DOMAIN`

Домен, который будет использоваться в настройках REALITY как маскировка. По умолчанию используется `dl.google.com`.

### `XRAY_PORT`

Порт, на котором будет слушать Xray. По умолчанию `8443`.

### `NGINX_DOMAIN`

Ваш домен для веб-сайта и получения сертификата Let's Encrypt. Если оставить пустым, `nginx` будет доступен только по HTTP.

### `LE_EMAIL`

Email для регистрации в Let's Encrypt. Если оставить пустым, выпуск сертификата пропускается.

## Что получится после установки

После успешного запуска будут созданы:

- `/opt/xray/docker-compose.yml`
- `/opt/xray/config.json`
- `/opt/nginx/docker-compose.yml`
- `/opt/nginx/conf.d/00-http.conf`
- `/opt/nginx/conf.d/10-https.conf` при успешном выпуске сертификата
- `/opt/www/html/index.html`

Контейнеры:

- `xray-reality`
- `nginx-web`
- `certbot` используется для выпуска сертификата

## Проверка и логи

Проверить контейнеры:

```bash
docker ps
```

Посмотреть логи Xray:

```bash
docker logs -f xray-reality
```

Посмотреть логи Nginx:

```bash
docker logs -f nginx-web
```

## Ограничения

- Скрипт явно ориентирован на Ubuntu и использует установку Docker через репозиторий Docker для Ubuntu.
- Внутри есть команды `apt-get`, `systemctl`, `ufw`, `firewall-cmd`, `iptables`, поэтому на других дистрибутивах поведение не гарантировано.
- Блок `ensure_prereqs` сейчас фактически ничего не устанавливает, потому что команды установки зависимостей в нем закомментированы.

## Пример запуска без вопросов

Можно передать значения через переменные окружения:

```bash
sudo REALITY_DOMAIN=dl.google.com \
XRAY_PORT=8443 \
NGINX_DOMAIN=example.com \
LE_EMAIL=admin@example.com \
bash install.sh
```

## Итог

Это не универсальный установщик VPN, а практический скрипт для быстрого разворачивания Xray REALITY и простого `nginx`-сайта на сервере. Он удобен для быстрого старта, если вы понимаете, что он меняет сетевые порты, firewall, Docker и конфигурацию веб-сервера на машине.
